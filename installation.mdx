## Prerequisites

Before you begin, ensure you have the following installed on your Windows machine:

<Steps>
  <Step title="Visual Studio">
    Install Visual Studio 2017 or later with the **.NET desktop development** workload.
    
    <Note>
      Visual Studio Community Edition is free and fully supports this project.
    </Note>
  </Step>

  <Step title=".NET Framework 4.7.2">
    The project targets **.NET Framework 4.7.2**. This is typically included with Visual Studio, but you can verify or install it separately:
    
    - Check your installed version in Visual Studio under **Tools > Get Tools and Features**
    - Download directly from [Microsoft .NET Framework 4.7.2](https://dotnet.microsoft.com/download/dotnet-framework/net472)
  </Step>

  <Step title="Git (Optional)">
    Install Git for Windows to clone the repository:
    
    ```bash
    # Download from:
    # https://git-scm.com/download/win
    ```
  </Step>
</Steps>

## System Requirements

- **Operating System**: Windows 7 SP1 or later
- **Processor**: 1.8 GHz or faster
- **RAM**: 2 GB minimum (4 GB recommended)
- **Disk Space**: 500 MB for the application and dependencies

## Clone the Repository

Clone the Validador de Acceso repository to your local machine:

```bash
git clone <repository-url>
cd ValidadorDeAcceso
```

<Note>
  If you don't have Git installed, you can download the source code as a ZIP file from your repository hosting service.
</Note>

## Project Structure

After cloning, you'll find the following structure:

```
ValidadorDeAcceso/
├── ValidadorDeAcceso/
│   ├── Form1.cs              # Main UI form
│   ├── Form1.Designer.cs     # UI component definitions
│   ├── Form1.resx           # Form resources
│   ├── GestorAcceso.cs      # Access management logic
│   ├── Usuario.cs           # User entity class
│   ├── Program.cs           # Application entry point
│   ├── ValidadorDeAcceso.csproj  # Project configuration
│   └── Properties/
│       ├── AssemblyInfo.cs
│       ├── Resources.resx
│       └── Settings.settings
└── ValidadorDeAcceso.sln    # Solution file
```

## Open in Visual Studio

<Steps>
  <Step title="Launch Visual Studio">
    Open Visual Studio on your machine.
  </Step>

  <Step title="Open the Solution">
    Navigate to **File > Open > Project/Solution** and select the `ValidadorDeAcceso.sln` file from the cloned repository.
  </Step>

  <Step title="Restore NuGet Packages">
    Visual Studio will automatically restore any required NuGet packages. Wait for the restore process to complete.
    
    <Note>
      This project uses standard .NET Framework libraries and doesn't require external NuGet packages.
    </Note>
  </Step>

  <Step title="Verify Build Configuration">
    Ensure the build configuration is set correctly:
    
    - **Configuration**: Debug or Release
    - **Platform**: Any CPU
    
    You can verify this in the toolbar at the top of Visual Studio.
  </Step>
</Steps>

## Verify Installation

To verify everything is set up correctly:

1. Check that the Solution Explorer shows all project files without errors
2. Verify that `ValidadorDeAcceso.csproj` targets **.NET Framework 4.7.2**:

```xml
<TargetFrameworkVersion>v4.7.2</TargetFrameworkVersion>
```

3. Ensure all references are resolved (no yellow warning icons in Solution Explorer)

<Warning>
  If you see missing reference errors, right-click on the project and select **Restore NuGet Packages**, then rebuild the solution.
</Warning>
