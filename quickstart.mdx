## Build the Project

Once you have the project open in Visual Studio, follow these steps to build the application:

<Steps>
  <Step title="Select Build Configuration">
    Choose your build configuration from the toolbar:
    
    - **Debug**: Includes debugging symbols (recommended for development)
    - **Release**: Optimized for production deployment
  </Step>

  <Step title="Build the Solution">
    Build the project using one of these methods:
    
    - Press `Ctrl + Shift + B`
    - Navigate to **Build > Build Solution**
    - Right-click the solution in Solution Explorer and select **Build**
  </Step>

  <Step title="Verify Build Success">
    Check the **Output** window for build results. You should see:
    
    ```
    ========== Build: 1 succeeded, 0 failed, 0 up-to-date, 0 skipped ==========
    ```
    
    The executable will be created at:
    - Debug: `bin\Debug\ValidadorDeAcceso.exe`
    - Release: `bin\Release\ValidadorDeAcceso.exe`
  </Step>
</Steps>

<Note>
  The first build may take a few moments as Visual Studio compiles all source files and resources.
</Note>

## Run the Application

<Steps>
  <Step title="Start from Visual Studio">
    Launch the application:
    
    - Press `F5` to run with debugging
    - Press `Ctrl + F5` to run without debugging
    - Click the **Start** button in the toolbar
  </Step>

  <Step title="Application Window Opens">
    The main form will appear with the following interface elements:
    
    - **Nombre** (Name) text field
    - **Contraseña** (Password) text field
    - **Agregar** (Add) button
    - **Validar** (Validate) button
    - **Mostrar usuarios** (Show users) toggle button
    - Status label showing "Listo" (Ready)
    - Failed attempts counter showing "Intentos fallidos: 0/3"
  </Step>
</Steps>

## Understanding Brute-Force Protection

Validador de Acceso includes built-in brute-force protection:

<Warning>
  After **3 failed login attempts**, the system will lock for 5 seconds.
</Warning>

### How It Works

The `GestorAcceso` class tracks failed login attempts:

```csharp ValidadorDeAcceso/GestorAcceso.cs
public bool ValidarAcceso(string nombre, string contraseña)
{
    // Block validation if max attempts reached
    if (intentosFallidos >= MAX_INTENTOS)
    {
        return false;
    }

    foreach (var usuario in usuarios)
    {
        if (usuario.Nombre == nombre && usuario.Contraseña == contraseña)
        {
            intentosFallidos = 0; // Success: Reset counter
            return true;
        }
    }

    // Failure: Increment counter
    intentosFallidos++;
    return false;
}
```

### Lockout Behavior

When the system is locked:

1. **All controls are disabled** for 5 seconds
2. The status shows: `"BLOQUEADO: Espere 5 segundos..."`
3. After 5 seconds, the counter resets and controls re-enable
4. The user can attempt login again

This is implemented asynchronously in `Form1.cs` to prevent UI freezing:

```csharp ValidadorDeAcceso/Form1.cs
if (gestor.EstaBloqueado())
{
    lblEstado.Text = "BLOQUEADO: Espere 5 segundos...";
    DeshabilitarAcciones(); // Disable all buttons

    // Wait 5 seconds without freezing the window
    await Task.Delay(5000);

    gestor.ReiniciarIntentos(); // Reset internal counter
    HabilitarAcciones();        // Re-enable buttons

    ActualizarIntentos();
    lblEstado.Text = "Sistema desbloqueado. Intente de nuevo.";
}
```

## Core Architecture

Understand the key components:

### Usuario Class

Represents a user with credentials:

```csharp ValidadorDeAcceso/Usuario.cs
internal class Usuario
{
    private string nombre="";  
    private string contraseña="";

    public string Nombre
    {
        get { return nombre; }
        set { nombre = value; }
    }   
    
    public string Contraseña
    {
        get { return contraseña; }
        set { contraseña = value; }
    }

    public Usuario(string nombre, string contraseña)
    {
        this.nombre = nombre;
        this.contraseña = contraseña;
    }
}
```

### GestorAcceso Class

Manages user authentication and brute-force protection:

- **MAX_INTENTOS**: Constant set to 3 failed attempts
- **AgregarUsuario()**: Adds users to the system
- **ValidarAcceso()**: Validates credentials and tracks failures
- **EstaBloqueado()**: Checks if the system is locked
- **ReiniciarIntentos()**: Resets the failure counter

### Form1 Class

The main UI form that:

- Displays input fields and buttons
- Handles user interactions
- Shows real-time status updates
- Manages the lockout UI state

```text Scenario 3: Mixed Attempts
5. Failed attempts reset on success
```

</CodeGroup>

## Running the Executable Directly

After building, you can run the executable without Visual Studio:

```bash
# Navigate to the build output directory
cd ValidadorDeAcceso\bin\Debug

# Run the application
ValidadorDeAcceso.exe
```

<Note>
  The application requires .NET Framework 4.7.2 runtime to be installed on the target machine.
</Note>
