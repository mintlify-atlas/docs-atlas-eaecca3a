The `GestorAcceso` class is the core authentication manager that handles user registration, credential validation, and brute-force protection through failed login attempt tracking.

```csharp
{

    public int IntentosFallidos { get; }
    public int MaxIntentos { get; }
}
```

### ValidarAcceso

```csharp
public bool ValidarAcceso(string nombre, string contraseña)
```

Validates user credentials against the registered users and manages failed attempt tracking.

<ParamField path="nombre" type="string" required>
  The username to validate
</ParamField>

<ParamField path="contraseña" type="string" required>
  The password to validate
</ParamField>

**Returns:** `bool`
- `true` if credentials are valid (also resets failed attempts counter to 0)
- `false` if credentials are invalid or system is already blocked (increments failed attempts counter)

**Behavior:**
1. Returns `false` immediately if the system is already blocked (`intentosFallidos >= MAX_INTENTOS`)
2. Searches for matching username and password in the user list
3. On success: resets `intentosFallidos` to 0 and returns `true`
4. On failure: increments `intentosFallidos` and returns `false`

**Example:**
```csharp
bool isValid = gestor.ValidarAcceso("alice", "secure123");
if (isValid)
{
    Console.WriteLine("Access granted!");
}
else
{
    Console.WriteLine("Invalid credentials");
}
```

### ReiniciarIntentos

```csharp
public void ReiniciarIntentos()
```

Resets the failed login attempts counter to zero. Typically called after a timeout period when the system is blocked.

**Returns:** `void`

**Example:**
```csharp

```

### EstaBloqueado

```csharp
public bool EstaBloqueado()
```

Checks whether the system is currently blocked due to excessive failed login attempts.

**Returns:** `bool`
- `true` if `intentosFallidos >= MAX_INTENTOS` (system is blocked)
- `false` otherwise (system is available for login attempts)

**Example:**

## Usage Example

```csharp
using ValidadorDeAcceso;
using System;

// Initialize the access manager
var gestor = new GestorAcceso();

// Add users
gestor.AgregarUsuario(new Usuario("admin", "admin123"));
gestor.AgregarUsuario(new Usuario("user", "password"));

// Validate access
bool success = gestor.ValidarAcceso("admin", "wrongpass");
Console.WriteLine($"Login: {success}");  // false
Console.WriteLine($"Failed attempts: {gestor.IntentosFallidos}");  // 1

// Another failed attempt
gestor.ValidarAcceso("admin", "wrongpass2");
Console.WriteLine($"Failed attempts: {gestor.IntentosFallidos}");  // 2

// Third failed attempt
gestor.ValidarAcceso("admin", "wrongpass3");
Console.WriteLine($"Blocked: {gestor.EstaBloqueado()}");  // true

// System is now blocked
gestor.ValidarAcceso("admin", "admin123");  // Returns false even with correct password

// Reset after timeout
gestor.ReiniciarIntentos();
Console.WriteLine($"Blocked: {gestor.EstaBloqueado()}");  // false

// Now can login successfully
success = gestor.ValidarAcceso("admin", "admin123");
Console.WriteLine($"Login: {success}");  // true
Console.WriteLine($"Failed attempts: {gestor.IntentosFallidos}");  // 0 (reset on success)
```

## Security Features

- **Brute-force protection**: Blocks the system after 3 consecutive failed attempts
- **Automatic counter reset**: Failed attempts counter resets to 0 on successful login
- **Layered security**: Even with correct credentials, access is denied when blocked
- **Manual reset capability**: Allows controlled unblocking through `ReiniciarIntentos()`

## Notes

- The class maintains state across login attempts
- Failed attempts are consecutive - a successful login resets the counter
- The blocking mechanism is temporary and requires manual reset via `ReiniciarIntentos()`
- User credentials are stored in-memory and not persisted
- Password comparison is case-sensitive
