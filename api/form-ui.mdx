## Event Handlers

### btnAgregar_Click

```csharp
private void btnAgregar_Click(object sender, EventArgs e)
```

Handles the "Agregar usuario" button click to register a new user.

**Behavior:**
1. Retrieves and trims the username from `txtNombre`
2. Retrieves the password from `txtContraseña`
3. Validates that both fields are not empty
4. Creates a new `Usuario` and adds it via `gestor.AgregarUsuario()`
5. Adds the username to `listaNombres`
6. Updates the status label with success message
7. Clears input fields and focuses on `txtNombre`
8. Calls `ActualizarListaUsuarios()` to refresh the UI list

**Validation:**
- Empty fields: Displays "Nombre y contraseña obligatorios."

**Example Flow:**
```
User enters: nombre="alice", contraseña="pass123"
→ Clicks "Agregar usuario"
→ lblEstado shows: 'Usuario "alice" agregado.'
→ Fields cleared
```

### btnValidar_Click

```csharp
private async void btnValidar_Click(object sender, EventArgs e)
```

Handles the "Validar acceso" button click to authenticate a user. This is an async method.

**Behavior:**
1. Retrieves and trims the username from `txtNombre`
2. Retrieves the password from `txtContraseña`
3. Validates that both fields are not empty
4. Calls `gestor.ValidarAcceso()` to validate credentials
5. On success: Displays welcome message
6. On failure: Displays error message
7. Updates the failed attempts counter display
8. Checks if system is blocked via `gestor.EstaBloqueado()`
9. If blocked:
   - Displays "BLOQUEADO: Espere 5 segundos..."
   - Disables all controls via `DeshabilitarAcciones()`
   - Waits 5 seconds using `await Task.Delay(5000)`
   - Calls `gestor.ReiniciarIntentos()` to reset counter
   - Re-enables controls via `HabilitarAcciones()`
   - Updates status to "Sistema desbloqueado. Intente de nuevo."
   - Clears password field and sets focus

**Validation:**
- Empty fields: Displays "Ingrese sus datos."

**Example Flow:**
```
User enters: nombre="alice", contraseña="wrong"
→ Clicks "Validar acceso"
→ lblEstado shows: "Credenciales incorrectas."
→ lblIntentos shows: "Intentos fallidos: 1/3"

(After 3 failed attempts)
→ lblEstado shows: "BLOQUEADO: Espere 5 segundos..."
→ All controls disabled
→ (5 second delay)
→ Controls re-enabled
→ lblEstado shows: "Sistema desbloqueado. Intente de nuevo."
```

## Private Helper Methods

### DeshabilitarAcciones

```csharp
private void DeshabilitarAcciones()
```

Disables all interactive controls when the system is blocked.

**Disables:**
- `btnValidar`
- `btnAgregar`
- `txtNombre`
- `txtContraseña`
- `button1` (if not null)
- `lstUsuarios`

### HabilitarAcciones

```csharp
private void HabilitarAcciones()
```

Re-enables all interactive controls after the block timeout expires.

**Enables:**
- `btnValidar`
- `btnAgregar`
- `txtNombre`
- `txtContraseña`
- `button1` (if not null)
- `lstUsuarios`

### ActualizarIntentos

```csharp
private void ActualizarIntentos()
```

Updates the failed attempts label with current counter values.

**Updates:** `lblIntentos.Text` to format: `"Intentos fallidos: {current}/{max}"`

**Example:** "Intentos fallidos: 2/3"

### ActualizarListaUsuarios

```csharp
private void ActualizarListaUsuarios()
```

Refreshes the user list box with all registered usernames.

**Behavior:**
1. Calls `lstUsuarios.BeginUpdate()` for performance
2. Clears all existing items
3. Iterates through `listaNombres` and adds each to the list box
4. Calls `lstUsuarios.EndUpdate()` to refresh
5. If list is empty, hides the list box and resets button text

## User Interaction Flow

### Adding a User

1. User enters username in `txtNombre`
2. User enters password in `txtContraseña`
3. User clicks `btnAgregar`
4. System validates inputs
5. System adds user to `gestor` and `listaNombres`
6. System displays success message
7. System clears input fields
8. System updates user list if visible

### Validating Access

1. User enters username in `txtNombre`
2. User enters password in `txtContraseña`
3. User clicks `btnValidar`
4. System validates credentials
5. System displays success or failure message
6. System updates failed attempts counter
7. If 3 failures reached:
   - System blocks for 5 seconds
   - All controls disabled
   - After timeout, system resets and re-enables controls

### Viewing Users

1. User clicks `button1` ("Mostrar usuarios")
2. System displays `lstUsuarios` with all registered usernames
3. Button text changes to "Ocultar usuarios"
4. User can click a username to auto-populate `txtNombre`
5. User clicks button again to hide the list

## Notes

- The form uses async/await for the 5-second block timeout to prevent UI freezing
- Password field does not use `PasswordChar` property (shows plain text)
- The `listaNombres` list is separate from the `GestorAcceso` user list for UI purposes
- All controls are disabled during the block period to prevent interaction
- The form automatically focuses on appropriate fields after actions for better UX
