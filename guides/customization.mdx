## Security Settings

### Maximum Login Attempts

The maximum number of failed login attempts before the account is temporarily locked is controlled by the `MAX_INTENTOS` constant.

**Location**: `GestorAcceso.cs:13`

```csharp
public const int MAX_INTENTOS = 3;
```

<Steps>
  <Step title="Locate the Constant">
    Open `GestorAcceso.cs` and find the `MAX_INTENTOS` constant at line 13.
  </Step>
  
  <Step title="Modify the Value">
    Change the value to your desired number of attempts:
    
    ```csharp
    // Allow 5 failed attempts before locking
    public const int MAX_INTENTOS = 5;
    
    // Allow only 2 failed attempts (stricter security)
    public const int MAX_INTENTOS = 2;
    ```
  </Step>
  
  <Step title="Rebuild the Project">
    Press **F6** in Visual Studio or use the Build menu to recompile.
  </Step>
</Steps>

<Warning>
Setting `MAX_INTENTOS` to 1 or lower may create a poor user experience. Consider the balance between security and usability.
</Warning>

#### How It Works

The `GestorAcceso` class tracks failed login attempts:

```csharp
public bool ValidarAcceso(string nombre, string contraseña)
{
    // Check if already blocked
    if (intentosFallidos >= MAX_INTENTOS)
    {
        return false;
    }
    
    // Validate credentials
    foreach (var usuario in usuarios)
    {
        if (usuario.Nombre == nombre && usuario.Contraseña == contraseña)
        {
            intentosFallidos = 0; // Reset on success
            return true;
        }
    }
    
    // Increment on failure
    intentosFallidos++;
    return false;
}
```

### Lockout Duration

The temporary lockout duration after exceeding max attempts is controlled by `Task.Delay()` in the validation logic.

**Location**: `Form1.cs:102`

```csharp
await Task.Delay(5000); // 5 seconds
```

<Steps>
  <Step title="Locate the Delay">
    Open `Form1.cs` and navigate to the `btnValidar_Click` method around line 102.
  </Step>
  
  <Step title="Modify the Duration">
    Change the delay value (in milliseconds):
    
    ```csharp
    // 10 second lockout
    await Task.Delay(10000);
    
    // 30 second lockout
    await Task.Delay(30000);
    
    // 1 minute lockout
    await Task.Delay(60000);
    ```
  </Step>
  
  <Step title="Update User Feedback">
    Update the status message to match the new duration:
    
    ```csharp
    lblEstado.Text = "BLOQUEADO: Espere 10 segundos...";
    await Task.Delay(10000);
    ```
  </Step>
</Steps>

<Tip>
For production systems, consider implementing exponential backoff (increasing delay with each subsequent lockout) for enhanced security.
</Tip>

**Location**: `Form1.Designer.cs` - Add to `InitializeComponent()` method

## Best Practices

<AccordionGroup>
  <Accordion title="Use Constants for Configuration">
    Instead of hardcoding values, define constants at the top of your class:
    
    ```csharp
    private const int LOCKOUT_DURATION_MS = 5000;
    private const string LOCKOUT_MESSAGE = "Account locked. Please wait.";
    ```
  </Accordion>
  
  <Accordion title="Maintain Consistency">
    If you change language or terminology, update all related messages throughout the application for consistency.
  </Accordion>
  
  <Accordion title="Test Security Changes">
    After modifying security parameters like `MAX_INTENTOS` or lockout duration, thoroughly test the authentication flow to ensure it behaves as expected.
  </Accordion>
  
  <Accordion title="Document Your Changes">
    Keep a changelog or comments in the code describing why certain values were chosen, especially for security-related configurations.
  </Accordion>
</AccordionGroup>
